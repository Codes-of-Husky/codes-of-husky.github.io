如何在GitHub上建立一个自己的博客

利用GitHub Pages建立自己的个人博客，有很多的方法。其中较主流的方法包括Jekyll和Hexo。本站是使用的是Hexo。本文分两个步骤介绍利用Hexo建立博客的过程。

在GitHub上的操作

  1. 创建自己的GitHub账号

  2. 新建一个仓库，为了可以通过网址username.github.io访问自己的博客，需要将仓库的名字设定为username.github.io

  3. 在仓库username.github.io的setting选项卡中，开启GitHub Pages

至此，GitHub上的操作已经完成。

在本地主机上的操作

  1. 按照hexo官网的教程安装Hexo，在安装Hexo之前需要先安装Node.js和Git。打开自己想要安装Hexo的目录，打开电脑的命令行界面，运行下面的命令：

    npm install -g hexo-cli    % 安装Hexo

  2. 如果运行上述命令没有报错，那么Hexo已经安装成功，接下来可以通过以下的命令初始化Hexo：

    hexo init blog    % 初始化一个Blog
cd blog           % 切换到Blog根目录下
npm install       % 安装依赖项

至此Hexo已经安装成功，本地会生成一个blog目录。默认的bolg目录结构如下：

.
├── _config.yml    % 配置文件
├── package.json   % 应用程序数据
├── scaffolds      % 模板文件
├── source         % 源文件
|   ├── _drafts    % 草稿
|   └── _posts     % 展示
└── themes         % 存放主题的目录

后期可以通过更改_config.yml文件来配置站点信息

  1. 运行如下的命令运行一个本地服务：

    hexo server    % 运行Hexo服务

如果运行上述命令没有报错，则通过浏览器查看http://localhost:4000/会看到一个网页，这就是Hexo默认的网页。

  1. (option)Hexo提供了很多的主题供选择，只需要下载主题文件，然后放进theme文件夹内，并修改站点配置文件_config.yml应用新的主题。

  2. 接下来把本地的Hexo博客部署到GitHub上。首先安装git部署工具：

    npm install hexo-deployer-git --save

    更改blog文件夹里面的_config.yml文件的deploy部分：

    deploy:
    type: git
    repo: git@github.com:yourname/yourname.github.io.git
    branch: master

  3. 设置本地git bash与GitHub账号之间的秘钥。使用以下命令生成ssh key：

    ssh-keygen -t rsa -C "youremail@email.com"

    其中 youremail@email.com是github bash的邮箱。

  4. 切换到存放秘钥的目录，利用以下命令打印出公钥：

    cat id_rsa.pub

  5. 复制打印出来的公钥，粘贴到GitHub账号的setting->SSH keys。

  6. 运行以下命令一键部署本地Hexo到GitHub Pages：

    hexo g -d
