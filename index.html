如何提高打字速度

发表于 2019-03-19 | 分类于 技术
本文字数： 1k | 阅读时长 ≈ 2 分钟

作为一个经常需要写一些技术文档或者文章的知识分子，可以较快的打字无疑是一项很加分的技能。我们平常接触最多的输入语言中，比较多的就是中文和英文，这篇文章就英文和中文介绍一些可以提高打字速度的方法。

Touch Typing

首先打字是一种肌肉性质的训练，就像跑步和健身，是不能停的，也不是一天能练成的，因此推荐大家经常训练打字。

基础知识

  1. 打字首先需要记住键盘中的键位，即每一个按键对应的是哪一个字母，这里推荐按照标准键盘的键位来练习，标准键位如下图所示：

  2. 在开始打字的时候，手摆放在键盘上的位置也是有要求的，手的起始摆放位置如下图：

    键盘上的F和J键对应的就是左手和右手的食指放置的位置。按键上会有一个突起，为的就是可以快速定位手的摆放位置。

  3. 指法，即手指敲击键盘的方法，手指的摆放也是有讲究的，在打字的时候，十个手指都有各自的作用，具体见下图：

    其中相同颜色的按键由同一手指敲击。

在了解了上述的基础知识之后，接下来想要快速的打字，就只有辛苦训练了。其中上述的指法是必须要掌握的。我们在打字的时候除了关心键盘和指法之外，还需要关注的就是输入法。对于英文的输入法往往就是全键盘输入，对于中文则有全键盘输入、双拼和五笔输入法等。下面先介绍英文字符输入的练习。

测试

在练习之前，我们需要做一个自我评价，评价的目的是了解自己现在的打字速度。我们可以通过typingtest.com来进行测试。

英文

如果简单的使用全键盘输入英文字符，那么我们需要做的就是记忆键盘上按键的位置，并多加练习。在输入的时候唯一需要注意的就是不要低头看键盘。即使有输入法的协助，为了能够快速的打字也需要这样做。这里介绍一些好用的练习网站

  1. keybr.com：能够在线练习打字，账号登陆还能保存历史记录。其练习打字的方法智能且有效。
  2. typingclub.com：界面简洁，里面把学习的过程组织得非常系统。有六百多个小章节，每个针对键盘上不同区域间键，循序渐进，练习相应的指法，章节完成后还有速度、出错率的总结，可以看到自己每次的进步。
  3. typingtest.com：也有一些有关快速打字的小技巧。

中文

中文字符的输入则需要首先熟练掌握拼音。在熟练掌握拼音的情况下，如果已经经过了上面英文字符输入的训练，使用简单的拼音输入法已经没有问题了。不过由于中文字符的编码问题，除了拼音输入法外，还有双拼、五笔等一些其它的输入法，由于我没有用过，故不多做解释。

使用Hexo在GitHub上搭建个人博客

发表于 2017-05-01 | 分类于 技术 ， 工具
本文字数： 2.5k | 阅读时长 ≈ 4 分钟

使用GitHub提供的GitHub Pages功能在GitHub上搭建个人博客，有很多种方法。其中较主流的方法包括使用Jekyll或者Hexo。本站是使用的是Hexo。本文分两个步骤介绍使用Hexo在GitHub上搭建并使用个人博客的过程。

搭建博客

在GitHub网页上的操作

  1. 创建自己的GitHub账号

  2. 新建一个仓库，为了可以通过网址username.github.io访问自己的博客，需要将仓库的名字设定为username.github.io

  3. 在仓库username.github.io的setting选项卡中，开启GitHub Pages

    至此，在GitHub网页上的操作已经部分完成。

在本地主机上的操作

搭建本地Hexo博客

  1. 按照Hexo官网的教程安装Hexo，在安装Hexo之前需要先安装Node.js和Git。切换到自己想要安装Hexo的目录，在电脑的cmd窗口，运行下面的命令：

    npm install -g hexo-cli    % 安装Hexo

  2. 如果运行上述命令没有报错，那么Hexo已经安装成功，接下来可以通过在电脑的cmd窗口运行以下的命令初始化Hexo：

    hexo init blog    % 初始化一个Blog
cd blog           % 切换到Blog根目录下
npm install       % 安装依赖项

    至此Hexo已经安装成功，本地会生成一个blog目录。默认的bolg目录结构如下：

    .
├── _config.yml    % 站点配置文件
├── package.json   % 应用程序数据
├── scaffolds      % 模板文件
├── source         % 源文件
|   ├── _drafts    % 草稿
|   └── _posts     % 展示
└── themes         % 存放主题的目录

    后期可以通过更改_config.yml文件来配置站点信息，此文件被称作站点配置文件。

  3. 在电脑的cmd窗口，运行如下的命令运行一个本地服务：

    hexo server    % 运行Hexo服务

    如果运行上述命令没有报错，则通过浏览器查看http://localhost:4000/会看到一个网页，这就是Hexo默认的网页。

  4. (optional)Hexo提供了很多的主题供选择，只需要下载你喜欢的主题文件，然后放进theme文件夹内，并修改站点配置文件应用新的主题。主题文件里面也包含一个_config.yml文件，此文件被称作主题配置文件。想要自定义主题，则需要修改主题配置文件。这里需要注意：站点配置文件和主题配置文件虽然文件名都是_config.yml，但其存放位置不一样。本站使用的主题是NexT。

    至此，本地站点已经搭建完毕，接下来需要把本地站点部署到GitHub上。

把本地Hexo博客部署到GitHub上

  1. 首先安装Hexo的Git部署工具，在电脑的cmd窗口运行：

    npm install hexo-deployer-git --save

    更改blog文件夹里面的_config.yml文件的deploy部分：

    deploy:
    type: git
    repo: git@github.com:yourname/yourname.github.io.git
    branch: master

    利用此工具，Hexo会自动将生成的静态网页部署到Git上去。但是真正的部署还需要一个验证的步骤。

  2. 在电脑的Git Bash窗口，使用以下命令生成SSH Key以设置本地Git Bash与GitHub账号之间的秘钥：

    ssh-keygen -t rsa -C "youremail@email.com"

    在设置密钥之前需要先设置本地Git Bash，填写自己的用户名和邮箱，如果已经填写则不用。其中youremail_at_email.com是github bash的邮箱。运行上述命令后得到了两个文件：id_rsa和id_rsa，存放于目录C:\Users\Administrator\.ssh下。

  3. 切换到存放秘钥的目录，在电脑的Git Bash窗口，利用以下命令打印出公钥：

    cat id_rsa.pub

  4. (在GitHub网页上执行的操作)复制打印出来的公钥，粘贴到GitHub账号的setting->SSH Keys。

  5. 在电脑的cmd窗口，运行以下命令生成Hexo站点：

    hexo generate       % 生成静态站点

  6. 在电脑的cmd窗口，运行以下命令部署本地Hexo到GitHub Pages：

    hexo deploy    % 部署到GitHub Pages

至此，使用Hexo在GitHub上搭建个人博客的操作已经完成，通过网址username.github.io就可以访问刚刚搭建的博客

使用博客

使用博客的方法较为简单，毕竟选择使用Hexo在GitHub上搭建个人博客的原因就是操作简单而且专注于内容。

  1. 在电脑的cmd窗口运行命令：

    hexo new "new_blog"    % 新建博客

    新建一个题目为new_blog的博客，源文件new_blog.md存放在blog\source\_posts\文件夹下。使用Markdown编辑器在new_blog.md里面填写博客内容并排版达到自己期望的效果，保存文件。

  2. 在电脑的cmd窗口运行以下命令以生成站点：

    hexo generate

  3. 在电脑的cmd窗口运行以下命令以部署站点：

    hexo deploy

到这里，关于个人博客的搭建和基础使用部分已经介绍完毕🎉。
